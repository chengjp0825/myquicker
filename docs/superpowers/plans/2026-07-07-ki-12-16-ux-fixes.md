# KI-12~16 High UX 缺陷修复计划

> 修复 5 项 High UX 缺陷，涉及 MainWindow/WakeOrchestrator/ScreenshotWindow/NativeMethods。
> 设计决策已确认：KI-12 低级键盘钩子、KI-13 SetWinEventHook、KI-16 ShowAt 重锚。

## 1. KI-14：菜单偏移 12px 裁切（最简，先做）

**根因**：`MainWindow.Width = menu.Width + 24`，`RootBorder Margin=12`，窗口左在内容左 12px。`WakeOrchestrator.ClampToScreen` 用 `MenuWidth/2` 算 `left = dipX - halfW`（内容左），`WakeUp` 设 `Left = location.X`（窗口左=内容左），导致内容中心偏光标 +12px，且贴右/下边时内容超出 12px 被裁。

**修复**：`MainWindow.WakeUp` 里 `Left = location.X - 12`、`Top = location.Y - 12`（窗口左移 12，使内容左 = location.X，内容中心 = 光标，内容右 = location.X + MenuWidth <= screenRight 不裁）。12 = RootBorder Margin，加常量 + 注释。

## 2. KI-15：截图放大镜异常保护（最简）

**根因**：`ScreenshotWindow.UpdateMagnifierLoupe`（行 519+）每次 mousemove 分配 CroppedBitmap+CopyPixels；`_baseImage` 状态异常（OnClosed 竞态）抛异常 -> `App.DispatcherUnhandledException` 不处理 -> 进程终止，ClipCursor 仍激活。

**修复**：`UpdateMagnifierLoupe` 整个方法体包 try/catch，异常时 `Debug.WriteLine` + `MagnifierLoupe.Visibility = Collapsed`（降级隐藏，不崩）。

## 3. KI-16：可见态二次唤醒 ShowAt 重锚

**根因**：`WakeOrchestrator.OnWakeContext`（行 59）`State != Hidden` 直接 return，Opening(150ms)+Closing(120ms) 内二次唤醒无反馈。现有 `WakeOrchestratorTests` 断言"可见态忽略"为有意。

**修复**：
- `WakeOrchestrator.OnWakeContext`：`State == Visible` 时调 `_presenter.ShowAt(dipLocation)` 重锚 + 更新 `_lastWakeTime`，不忽略。`Opening/Closing` 态仍忽略（动画中重锚冲突）。
- `MainWindow.WakeUp`：开头检查 `_isAwake`，已可见时仅更新 `Left/Top`（瞬移），不重放动画（避免闪）。
- 更新 `WakeOrchestratorTests`：把"Visible 态二次唤醒被忽略"断言改为"Visible 态二次唤醒触发 ShowAt 重锚"。

## 4. KI-12：菜单键盘交互（低级键盘钩子）

**根因**：窗口 `WS_EX_NOACTIVATE` 不获系统键盘焦点，`PreviewKeyDown` 不触发，菜单无键盘响应。

**修复**：挂 `WH_KEYBOARD_LL` 低级键盘钩子，菜单可见时拦 Esc/Enter/方向键。

**实现**：
- `NativeMethods`：加 `WH_KEYBOARD_LL=13`、`HC_ACTION=0`、`KBDLLHOOKSTRUCT`、`WM_KEYDOWN=0x0100`/`WM_KEYUP`、`VK_ESCAPE/VK_RETURN/VK_LEFT/VK_RIGHT/VK_UP/VK_DOWN`、`LowLevelKeyboardProc` 委托。
- `MainWindow`：
  - 字段：`_keyboardHookId`、`_keyboardProc`、`_focusedButtonIndex`。
  - `OnSourceInitialized`：挂载键盘钩子（同鼠标钩子，需考虑 KI-24 时序--延后到 Background 或在 SourceInitialized 挂载，键盘钩子不像鼠标钩子那样会卡，但仍加异常保护）。
  - `OnClosed`：卸载键盘钩子。
  - 回调 `KeyboardHookCallback`：try/catch（同 KI-3 保护）；`!_isAwake` 放行；`WM_KEYDOWN` 时：
    - `VK_ESCAPE` -> `RaiseDismissRequested()`，吞键。
    - `VK_RETURN` -> 触发 `_focusedButtonIndex` 按钮点击，吞键。
    - `VK_LEFT/RIGHT/UP/DOWN` -> 调整 `_focusedButtonIndex`（Left/Right ±1，Up/Down ±columns），clamp，聚焦按钮，吞键。
    - 其他键放行。
  - 按钮列表：从 `ActionsControl.ItemContainerGenerator` 取，或维护独立列表。列数从 `_preferences.Menu.GridColumns`（2 或 3）。
  - 唤醒时（`WakeUp`）：`_focusedButtonIndex = 0`，聚焦第一个按钮（`button.Focus()` WPF 逻辑焦点，NOACTIVATE 窗口内可用）。
  - 视觉高亮：复用按钮的 FocusVisualStyle，或手动设边框。

## 5. KI-13：Alt-Tab/失活关闭菜单（SetWinEventHook）

**根因**：`WS_EX_NOACTIVATE` 窗口不激活，`Deactivated` 不触发。Alt-Tab/Win+D 切换前台时，topmost 菜单滞留新前台之上。

**修复**：`SetWinEventHook(EVENT_SYSTEM_FOREGROUND)` 监听前台窗口变化，菜单可见时前台变了就 Dismiss。

**实现**：
- `NativeMethods`：加 `SetWinEventHook`、`UnhookWinEvent`、`EVENT_SYSTEM_FOREGROUND=3`、`WINEVENT_OUTOFCONTEXT=0`、`WinEventProc` 委托。
- `MainWindow`：
  - 字段：`_foregroundHook`、`_winEventProc`。
  - `OnSourceInitialized`：挂载 `SetWinEventHook(EVENT_SYSTEM_FOREGROUND, ...)`。
  - `OnClosed`：`UnhookWinEvent`。
  - 回调 `OnForegroundChanged`：`_isAwake` 时 `RaiseDismissRequested()`。回调在挂载线程消息循环（主 UI 线程），可直接访问，但用 `Dispatcher.Invoke` 防御。
  - 注意：菜单自身 NOACTIVATE 不触发前台变化，只有真正 Alt-Tab/切换才触发。

## 风险与注意

- **KI-12 键盘钩子时序**：参考 KI-24，低级钩子挂载后主线程同步阻塞会卡全局。键盘钩子挂载在 `OnSourceInitialized`（消息循环已运行），回调极速（`!_isAwake` 直接放行）。异常保护同 KI-3。
- **KI-12 焦点**：NOACTIVATE 窗口内 `Button.Focus()` 设 WPF 逻辑焦点，但系统键盘焦点不在窗口。键盘事件靠钩子拦截后手动处理，不依赖系统焦点。
- **KI-13 回调线程**：`WINEVENT_OUTOFCONTEXT` 回调在挂载线程消息循环（主 UI 线程），用 `Dispatcher.Invoke` 防御。
- **KI-16 测试更新**：现有测试断言"可见态忽略"，需改为"重锚"。可能有多个测试用例。
- **键盘钩子常驻 vs 按需**：本方案常驻（同鼠标钩子），`!_isAwake` 快速放行。若担心常驻开销，可改为唤醒挂载/关闭卸载（但增加时序复杂度，先用常驻）。
